# Security Policy

## Reporting Security Issues

If you discover a security vulnerability in XKernal, please report it responsibly by emailing **security@cognitivesubstrate.dev** instead of using the public issue tracker.

## Response Timelines

We commit to the following response and resolution timelines:

| Severity | Initial Response | Target Resolution |
|----------|-----------------|-------------------|
| Critical | 4 hours | 24 hours |
| High | 8 hours | 72 hours |
| Medium | 24 hours | 7 days |
| Low | 48 hours | 30 days |

## Supported Versions

Security updates are provided for the following versions:

| Version | Status | Support Until |
|---------|--------|---------------|
| v1.0.x | Actively Supported | Current |

Older versions are not supported. We recommend upgrading to the latest available version.

## Responsible Disclosure

We follow responsible disclosure practices:

1. **Report privately** to security@cognitivesubstrate.dev with detailed information about the vulnerability
2. **Allow time** for the team to investigate and develop a fix before public disclosure
3. **Coordinate timing** for vulnerability announcement and patch release
4. **Credit** will be given to researchers who follow this process in good faith

## Patch Releases

Security patches will be released as maintenance updates to supported versions. Critical issues may trigger emergency releases outside of the standard release cycle.

## Security Best Practices

Users of XKernal should:

- Keep the project updated to the latest version
- Monitor the security advisories on this repository
- Review dependencies for known vulnerabilities
- Report any suspicious activity or potential exploits

## Questions?

For security-related questions or concerns, contact security@cognitivesubstrate.dev
